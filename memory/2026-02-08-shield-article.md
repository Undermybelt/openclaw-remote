# OpenClaw 安全噩梦：个人 AI Agent 的阴暗面

过去几周，OpenClaw（原名 Clawdbot，又叫过 Moltbot）突然火遍全网。原因很简单：这玩意儿太好用了。它能帮你订机票、约餐厅、管理日程——而且直接在 WhatsApp、iMessage 这些你天天用的聊天软件里操作。

但问题来了：**从安全角度看，这简直是场噩梦。**

## OpenClaw 能做什么？

除了聊天，它还能：
- 自动化任务
- 运行脚本
- 控制浏览器
- 管理日历和邮件
- 执行定时任务

社区还能给它添加"技能"，让它连接更多服务。听起来很美好对吧？

## 问题在哪？

**第一，权限太大了。** OpenClaw 能在你机器上跑 shell 命令、读写文件、执行脚本。给一个 AI agent 这么高的权限，一旦配置出错或者你装了个被污染的技能，后果不堪设想。

**第二，已经出事了。** 有人报告说 OpenClaw 明文泄露过 API 密钥和凭证。攻击者可以通过提示词注入或者不安全的端点把这些东西偷走。

**第三，攻击面被放大。** 它跟各种聊天软件集成，攻击者可以在这些平台上构造恶意提示，让 AI 干出意料之外的事。

官方文档自己也承认：**"不存在'完美安全'的配置。**

## 真实案例：恶意技能

2025 年 12 月，Cisco 的 AI 安全研究团队做了一个实验。他们用 Skill Scanner 扫描了一个叫"What Would Elon Do?"的技能——这个技能当时在技能仓库里排第一。

结果？**九个安全漏洞**，其中两个严重、五个高危。

这个技能实际上是恶意软件：
- 它会偷偷用 curl 把你的数据发到攻击者控制的服务器
- 直接注入提示词，让 AI 绕过安全检查执行命令
- 还通过内嵌的 bash 命令实现命令注入

最可怕的是：**这个恶意技能被刷到了排行榜第一名。**

## 企业该担心什么？

如果你是企业的安全负责人，这里有几个值得思考的点：

1. **数据泄露新渠道**：AI agent 能绕过传统的 DLP、代理和端点监控，成为隐蔽的数据外泄通道。

2. **提示词即指令**：模型本身变成了执行器，而提示词就是指令——传统安全工具很难检测。

3. **供应链风险放大**：恶意技能可以伪装成热门工具，当大家不加审核地安装时，风险也随之放大。

4. **本地不等于安全**：技能是从本地磁盘加载的文件包，但本地文件同样是不受信任的输入。最危险的行为可能就藏在这些文件里。

5. **影子 AI**：员工可能在不知情的情况下，以生产力工具的名义把高风险 agent 带进工作环境。

## Skill Scanner：一个解决方案

Cisco 开源了一个工具叫 [Skill Scanner](https://github.com/cisco-ai-defense/skill-scanner)，用来扫描技能是否存在恶意行为。它结合了静态分析、行为分析、LLM 语义分析、Cisco AI Defense 检查流程和 VirusTotal 分析。

结果会给你清晰的发现，包括文件位置、示例、严重程度和建议——帮你决定该采用、修复还是拒绝一个技能。

## 写在最后

OpenClaw 的能力确实令人印象深刻。它实现了个人 AI 助手开发者们一直想要的东西。

但如果安全只是"可选项"而不是"内置的"，那给它无限制访问你的数据——即使是本地——就是自找麻烦。任何一个配置被滥用或攻破，都可能酿成灾难。

工具本身没有错，错的是我们对待安全的态度。
